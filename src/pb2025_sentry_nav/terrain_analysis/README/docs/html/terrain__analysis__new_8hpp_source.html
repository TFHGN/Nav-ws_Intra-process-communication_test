62 /**
63  * @class TerrainAnalysisNode
64  * @brief 地形分析节点类 - 用于实时地形评估和可通行性分析
65  *
66  * @details
67  * 该类实现了基于点云的地形分析功能，通过维护一个滚动的体素网格来
68  * 跟踪机器人周围的地形信息。主要特性包括：
69  *
70  * **数据流程**：
71  * 1. 接收激光雷达点云 (registered_scan)
72  * 2. 接收里程计位姿 (lidar_odometry)
73  * 3. 进行体素化与地形网格更新
74  * 4. 计算地面高程与障碍物分布
75  * 5. 发布带高程信息的地形图 (terrain_map)
76  *
77  * **处理模式**：
78  * - 100Hz 定时器驱动主处理循环
79  * - 回调函数仅更新缓冲区标志
80  * - 所有计算在定时器回调中完成
81  *
82  * **坐标系**：
83  * - 输入/输出均使用 "odom" 坐标系
84  * - 相对于机器人基座的地形评估
85  *
86  * @note 该节点设计为可组合组件，可加载到 Nav2 容器中
87  */
88 class TerrainAnalysisNode : public rclcpp::Node

91  /**
92  * @brief 构造函数 - 初始化地形分析节点
93  *
94  * @param options ROS2 节点选项，用于配置节点行为
95  *
96  * @details
97  * 构造函数执行以下初始化步骤：
98  * 1. **参数声明与加载**：声明并读取所有配置参数
99  * 2. **体素网格初始化**：创建地形体素云数组
100  * 3. **订阅器创建**：
101  * - lidar_odometry：接收机器人位姿
102  * - registered_scan：接收配准后的点云
103  * - joy：接收手柄清空指令
104  * - map_clearing：接收地图清空指令
105  * 4. **发布器创建**：terrain_map - 发布带高程的地形图
106  * 5. **定时器创建**：100Hz 主处理循环
107  *
108  * @note 使用 SensorDataQoS 以支持高频传感器数据
109  * @warning 构造函数设计为非阻塞，适配组件容器环境
110  */
111  explicit TerrainAnalysisNode(const rclcpp::NodeOptions &options = rclcpp::NodeOptions());

140  * 1. **时间戳管理**：提取点云时间戳，初始化系统时钟

147  *
148  * @note 裁剪后的点云存储在 laserCloudCrop 中
149  * @note 采用动态高度范围（与距离成正比）以适应地形起伏

182  /**
183  * @brief 定时器回调函数 - 主处理循环（100Hz）
184  *
185  * @details
186  * 该函数是地形分析的核心处理逻辑，每10ms执行一次，包含以下步骤：
187  *
188  * **1. 体素网格滚动更新**：
189  * - 根据机器人移动动态调整地形网格原点
190  * - 移出旧体素，清空并重用内存
191  * - 保持机器人始终在网格中心区域
192  *
193  * **2. 点云堆叠与体素化**：
194  * - 将新点云按位置映射到对应体素
195  * - 累积多帧点云以提高密度
196  * - 记录体素更新次数和时间
197  *
198  * **3. 体素降采样与衰减**：
199  * - 对达到更新阈值的体素执行降采样
200  * - 应用时间衰减规则移除过期点
201  * - 根据距离和时间保留有效数据
202  *
203  * **4. 地面高程估计**：
204  * - 将点云投影到平面体素网格
205  * - 使用分位数或最小值法估计地面高度
206  * - 限制地面抬升以过滤异常值
207  *
208  * **5. 动态障碍物过滤**（可选）：
209  * - 检测高于地面且在特定视场角内的点
210  * - 累积障碍物点数判断是否为动态物体
211  * - 过滤掉可能的移动障碍物
212  *
213  * **6. 可通行性评估**：
214  * - 计算点相对于地面的高度差
215  * - 过滤出车辆高度范围内的障碍
216  * - 标记无数据区域为潜在障碍
217  *
218  * **7. 地形图发布**：
219  * - 将带高程信息的点云转换为ROS2消息
220  * - 设置正确的时间戳和坐标系
221  * - 通过 terrain_map 话题发布
222  *
223  * @note 仅在 new_laser_cloud_ 为 true 时执行处理
224  * @note 所有计算均在 "odom" 坐标系下进行
225  * @warning 确保处理时间 < 10ms 以避免定时器堆积
226  */
227  void timerCallback();
